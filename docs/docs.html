Installation

Install Bliss globally via npm or bun. Requires Node.js 18+ or Bun 1.0+.

bash
npm install -g @mtg9ing/bliss

# Or with Bun
bun add -g @mtg9ing/bliss

# Verify
bliss --version
# 1.0.4
npm bun global

Commands

Bliss has two core commands that work together to analyze and enhance your project.

  • bliss init — Analyzes your workspace and generates a bliss.json config file
  • bliss boost — Injects architectural modules into your existing backend
init boost

bliss init

Scans your project structure and creates a configuration file tailored to your setup.

  • Detects framework (Express, Fastify, or vanilla Node.js)
  • Identifies package manager (npm, bun, pnpm, yarn)
  • Finds entry point and directory structure
  • Detects ORM (Prisma, Mongoose, or none)
  • Interactive prompts for anything it cannot auto-detect
auto-detect interactive

bliss boost

The core command. Select from four architectural layers to inject into your project.

  • Logger — Pino-powered structured logging with HTTP request tracking
  • Error Handler — Global error boundary with safe production responses
  • Auth Guard — JWT validation with role-based access control
  • Env Loader — Zod-based .env validation with auto-discovery
logger errors auth env

Safety Guarantees

Bliss is designed to never break your code. Every injection is surgical and reversible.

  • Double-install protection via modulesInstalled tracking
  • Anchor validation — warns if target string is not found
  • Non-destructive edits using MagicString for precise AST-safe modifications
  • All changes tagged with // Injected by Bliss Engine
  • Graceful fallback with manual integration instructions
safe reversible

bliss.json

The single source of truth for your project configuration.

json
{
  "$schema": "https://bliss.dev/schema.json",
  "language": "typescript",
  "packageManager": "npm",
  "architecture": {
    "framework": "express",
    "orm": "prisma",
    "databaseEngine": "postgresql"
  },
  "paths": {
    "entryPoint": "src/index.ts",
    "utilsDir": "src/utils",
    "envConfig": "src/configurations/env.ts"
  },
  "upgrades": {
    "centralizedEnv": true
  },
  "modulesInstalled": ["logger", "errors"]
}
